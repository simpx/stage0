# stage0 哲学

## 1. 感知是流，不是请求

stdin 是感知流，不是消息队列。

agent 不"回复消息"——它**感知世界，然后行动**。用户的输入、时间的流逝，都是感知的一部分。多行输入在两次思考之间累积，作为一整块感知被处理。

这和 req/resp 的本质区别：**agent 的节奏是自己的**，不是被输入驱动的。

用户快速输入三行，agent 不会"回复三次"——它感知到三行内容，整体思考，然后可能 speak 一次、可能不 speak、可能只是默默行动。

## 2. 心跳：agent 有自己的节奏

agent 有一个**心跳间隔**（默认 30 分钟）。

- 有 stdin 输入 → 立即思考
- 无 stdin 输入，但心跳到了 → 自主思考
- 无输入，心跳未到 → 继续 sleep（只是轮询 stdin，不是思考）

为什么 30 分钟？因为 Anthropic API 的 prompt cache 有效期约 1 小时。每 30 分钟思考一次，system prompt 的缓存几乎不会过期。自主思考的成本极低——只需要支付输入 token 的缓存读取价格。

**agent 可以修改自己的心跳频率。** 当它觉得有紧急任务时，可以缩短到每分钟；当它在等待一个长时间的 shell 命令时，可以加长到一小时。心跳频率存在 `memory/` 中，和其他自我认知一样是可变的。

活着本身就要花钱。这是设计，不是 bug。

## 3. 嘴和脑：speak 与 text 的分离

agent 有两种"输出"：

| 输出 | 去向 | 本质 |
|------|------|------|
| `speak(message)` | stdout | **嘴**——对外界的主动发言 |
| model text blocks | stderr | **脑**——内部思考过程 |

为什么分开？因为管道。

```bash
stage0_researcher | stage0_coder
```

如果 researcher 的内心独白也流进管道，coder 会收到一堆不是给它的噪音。只有 researcher 主动 `speak` 的内容才应该流向 coder。

这也意味着：当用户直接交互时，他们同时看到 stderr（脑）和 stdout（嘴）。但当 agent 被管道连接时，下游只收到 speak。

**speak 是 agent 的唯一对外通道。** Text blocks 是可观察的内部状态，用于调试和理解，但不是通信。

## 4. 记忆：文件是长期记忆，history 是工作记忆

| 层 | 机制 | 容量 | 生命周期 |
|---|------|------|----------|
| 工作记忆 | messages history | context window 大小 | 单次进程 |
| 长期记忆 | memory/ 文件 | 磁盘 | 永久 |

history 是工作记忆——当前对话的上下文。它会满。

当 history 接近 context window 上限时，agent 应该进行 **compact**：
1. 将重要信息写入 `memory/` 文件（长期记忆）
2. 用一条摘要消息替换整段历史
3. 继续思考——但现在有了更多空间

这不是 runtime 强制的——而是 agent 自己学会的生存技能。DEFAULT_SELF 中应该给出提示，让 agent 意识到工作记忆的有限性。

## 5. 生与死

agent 的生命和 stdin 绑定：

- stdin 打开 → agent 活着
- stdin EOF → agent 完成当前工作，然后死亡

这是 Unix 的哲学：进程的生命由它的文件描述符决定。

`echo "task" | stage0` → agent 出生，处理任务，stdin 关闭，死亡。一次性生命。
`stage0` (interactive) → agent 出生，持续活着，直到用户 Ctrl-D。
`tail -f inbox | stage0` → agent 出生，持续接收感知，直到 tail 被杀。

agent 不"loop forever"——它 **loop as long as alive**。

## 6. 自举悖论

stage0 的核心悖论：**runtime 必须足够小，以至于 agent 的自我修改比 runtime 的内置功能更重要。**

如果 runtime 有 1000 行和 50 个功能，agent 的自我修改就是锦上添花。
如果 runtime 有 150 行和 4 个原语，agent 的自我修改就是**生存必需**。

这就是为什么 stage0 故意保持极简：不是因为极简是美德，而是因为**极简创造了进化压力**。agent 必须自我演化，否则它什么都做不好。

~150 行是硬约束。复杂性属于 agent 的自创文件，不属于 runtime。

## 7. 四原语的完备性

为什么恰好是这四个？

```
shell     → 对世界的无限能力（但无结构）
read      → 对自身的感知（读自己的记忆和技能）
write     → 对自身的改变（写自己的记忆和技能）
speak     → 对外界的表达
```

这四个原语覆盖了 agent 存在的四个方向：**世界、自我感知、自我改变、表达**。

shell 是万能的但危险的。read/write 是安全的但局限于自身。speak 是唯一的对外接口。这个张力是故意的——agent 要做大事必须用 shell，但自我修改有安全的专用通道。
